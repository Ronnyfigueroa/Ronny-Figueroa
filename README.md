Objetivo del sistema

Desarrollar un programa en Python que simule el juego "Piedra, Papel o Tijera", permitiendo al usuario jugar contra la computadora mediante una selección aleatoria. El sistema debe validar la entrada del usuario, determinar el resultado de cada partida y aplicar los conocimientos adquiridos durante el semestre en la asignatura de Lógica de Programación.

Descripción de funcionalidades

El sistema permite al usuario ingresar una de las tres opciones disponibles: piedra, papel o tijera. Posteriormente, la computadora selecciona una opción de forma aleatoria y el programa compara ambas elecciones para determinar si el usuario ganó, perdió o empató. Además, incorpora una validación de entrada mediante un bucle, evitando que el usuario continúe hasta ingresar una opción válida.

Introducción

La programación permite desarrollar soluciones que automatizan tareas y resuelven problemas mediante algoritmos. Como parte del aprendizaje de la asignatura, se desarrolló un proyecto basado en el juego "Piedra, Papel o Tijera", el cual integra diversos conceptos estudiados durante el semestre. Este proyecto representa una aplicación práctica de los fundamentos de programación, permitiendo fortalecer habilidades relacionadas con el análisis de problemas, diseño de algoritmos e implementación de programas en Python.

Descripción del problema

El problema consiste en desarrollar un programa capaz de simular el juego "Piedra, Papel o Tijera" entre un usuario y la computadora. El sistema debe recibir la elección del jugador, generar una selección aleatoria para la computadora, validar que la opción ingresada sea correcta y determinar automáticamente el resultado del juego según las reglas establecidas.

Relación con los contenidos de la asignatura

El desarrollo del proyecto permitió aplicar los contenidos abordados durante el semestre. Inicialmente se realizó el análisis del problema y la definición de los objetivos del sistema. Posteriormente se utilizaron variables, algoritmos y pseudocódigos para planificar la solución. Durante la implementación se emplearon estructuras de decisión, operadores lógicos y condicionales para establecer las reglas del juego. Finalmente, se incorporó una lista para almacenar las opciones disponibles y un bucle con condición para validar la entrada del usuario, complementando el proyecto con diagramas de casos de uso, arquitectura de software y diagrama de flujo.

Explicación del sistema desarrollado

El sistema fue desarrollado en Python utilizando Visual Studio Code como entorno de programación. El programa comienza mostrando el título del juego y solicita al usuario que ingrese una opción. Mediante un bucle de validación, verifica que la entrada corresponda a una de las opciones permitidas. Una vez validada, la computadora selecciona aleatoriamente una jugada utilizando el módulo random. Finalmente, el programa compara ambas elecciones mediante estructuras condicionales y operadores lógicos para mostrar si el resultado es una victoria, un empate o una derrota para el jugador.

Cronograma de desarrollo

**Semana 1:** Se realizó el análisis del problema, identificando los diferentes tipos de problemas y seleccionando el proyecto "Piedra, Papel o Tijera". Además, se definió el objetivo principal del sistema y el problema que se buscaba resolver mediante la programación.

**Semana 2:** Se inició el aprendizaje del entorno de desarrollo Visual Studio Code y del lenguaje Python. Durante esta etapa se estudiaron los tipos de errores, especialmente los errores de sintaxis, y las buenas prácticas para prevenirlos durante el desarrollo del proyecto.

**Semana 3:** Se estudiaron las variables y la elaboración de diagramas. Como parte del proyecto se diseñó el diagrama de casos de uso y el diagrama de arquitectura de software, con el propósito de representar la interacción del usuario con el sistema y la estructura general de la aplicación.

**Semana 4:** Se trabajó en el diseño del algoritmo que daría solución al problema. Se elaboraron pseudocódigos como guía para organizar la lógica del programa y posteriormente se desarrolló una primera versión del código en Python para iniciar la implementación del proyecto.

**Semana 5:** Se aplicaron las estructuras de decisión, los operadores lógicos y las sentencias condicionales para implementar las reglas del juego. Estas herramientas permitieron que el programa determinara correctamente cuándo el usuario gana, pierde o empata frente a la computadora.

**Semana 6:** Se elaboró el primer diagrama de flujo y se desarrolló una versión funcional del programa, incorporando la lógica principal del juego. Además, se creó el repositorio del proyecto en GitHub para documentar y gestionar el desarrollo del sistema.

**Semana 7:** Se incorporó el uso de listas y un bucle con validación para controlar que el usuario ingresara únicamente opciones válidas. También se actualizó el diagrama de flujo y el código del programa. Finalmente, se realizaron pruebas de funcionamiento para verificar que el sistema respondiera correctamente en todos los escenarios posibles y se corrigieron los errores encontrados antes de finalizar el proyecto.
